# Лабораторная работа №3 студента ИИКС ПМИ первого курса Сарычева Владислава

Данная лабораторная работа представляет собой модель по работе с двусвязным деком. В рамках проекта реализованы два типа сортировок, инструментарий для работы с деком, позволяющий работать с ним как со списком:

1. Создание дека
2. Получение текущего размера
3. Добавление элемента в произвольное место дека
4. Удаление произвольного элемента дека
5. Перестановка элементов в деке
6. Конвертация дека в список и обратно
7. Очистка дека
8. Получение указателей на начало и конец

Проведено сравнение временных затрат представленных сортировок и построены графики зависимости времени их работы от размеров сортируемых деков
# Структура проекта
## Дек
Дек реализуется на основе двусвязного списка. Элементы дека являются структурами, хранящими данные о домах. Структура данных имеет вид:
* Название застройщика:строка
* Название микрорайона:строка
* Тип: перечисление (панельный, кирпичный,монолитный)
* Год постройки:четырехзначное число
* Наличие лифта:логическое значение
* Наличие мусоропровода: логическое значение
* Количество квартир:целое число
* Количество этажей:целое число
* Средняя площадь квартиры: вещественное число
## Код

Программа может работать в трех режимах:

1. Генерация случайного дека:
* Программа, запущенная с флагом--generate N или-g N, генерирует случайные данные в соответствии со схемой данных в количестве N строк. 
* Если указан флаг--out="output.csv" или-o output.csv, вывод осуществляется в соответствующий файл а формате
значений, разделенных запятой (CSV). Иначе вывод осуществляется на стандартный поток вывода в соответствующем формате.
2. Сортировка дека:
* Программа, запущенная со флагом--sort или-s, считывает данныев соответствии со схемой данных в структуру-контейнер, сортирует их и выводит отсортированные. 
* Если указан флаг--out="output.csv" или-o output.csv, вывод осуществляется в соответствующий файл а формате значений, разделенных запятой (CSV). Иначе вывод осуществляется на стандартный поток вывода в соответствующем формате.
* Если указан флаг--in="data.csv"или-i data.csv, ввод осуществляется из соответствующего файла. Иначе ввод осуществляется со стандартного потока ввода в соответствующем формате. По-умолчанию сортировка осуществляется по возрастанию. Также по возрастанию следует сортировать, если указан флаг--type=asc или-t. Если указан флаг--type=desc или-t D, сортировка выполняется в порядке убывания

3. Вывод дека:
* Программа, запущенная с флагом–print или-P, считывает данные и выводит их в формате таблицы с фиксированной шириной.
* Если указан флаг--in="data.csv"или-i data.csv, ввод осуществляется из соответствующего файла. Иначе со стандартного потока ввода
считывается имя файла, содержащее данные для считывания.
* Если указан флаг--out="output.txt" или-o output.txt, вывод осуществляется в соответствующий файл. Иначе вывод осуществляется на
стандартный поток вывода.

## Сортировки

* В проекте реализованы гномья сортировка и сортировка Хоары. Приведем графики зависимости времени работы сортировок в зависимости от размера сортируемого массива:
> [Гномья сортировка](gnomsorted_final.ods)
> [Сортировка Хоары](quick_final.ods)
* Как можно заметить из приложенных графиков гномья сортировка работает быстрее быстрой, хотя исходя из матожидания времени работы ситуация должна быть обратной. Все дело в том, что быстрая сортировка задействует работу с не соседними элементами, что требует линейного пробегания по деку. При большом кол-ве повторов это дает сильный прирост во времени работы, хоть и сохраняет главный член в асимптотике. В свою очередь гномья сортировка работает исключительно с соседними элементами, что значительно уменьшает необходимое кол-во операций, а это в свою очередь кратно уменьшает время работы алгоритма.
